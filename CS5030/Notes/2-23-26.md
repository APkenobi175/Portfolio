## MPI

- MPI stands for Message Passing Interface
- Common practice to identify processes by ID
- There are multiple MPI packages, and the one we will use is called OpemMPI, which has nothing to do with openMP lol. 
- OpenMP is integrated into gcc now so you don't have to do anything special to use it tbh.

- Other implementations include MPICH, and MVAPICH, which are also popular MPI implementations.

- You need an `MPI_Init` function call to initialize the MPI environment, and then you can use other MPI functions to send and receive messages between processors.

  - This is where you start using MPI

- `MPI_Finalize` is used to clean up the MPI environment when you are done using it.

- `MPI_Recv` is used to receive a message from another processor, and `MPI_Send` is used to send a message to another processor.

- `MPI_Comm_rank(MPI_COMM_WORLD, &my_rank)` is used to get the rank of the current process. The rank is a unique identifier for each process, and it is used to determine which process is which when sending and receiving messages.

- `MPI_Comm_size(MPI_COMM_WORLD, &num_procs)` is used to get the total number of processes in the MPI environment. This is useful for determining how many processes are available to send and receive messages.

- What is MPI_COMM_WORLD?
  - Its called a communicator, and it is a group of processes that can communicate with each other. MPI_COMM_WORLD is the default communicator that includes all the processes in the MPI environment.
  - Is this like the interconnect? no the communicator uses the interconnect to send messages between processes, but it is not the interconnect itself. The interconnect is the hardware that allows the processes to communicate with each other, while the communicator is a software construct that defines which processes can communicate with each other.

### MPI Datatypes

- MPI provides a set of predefined datatypes that can be used to specify the type of data being sent or received in MPI communication functions. These datatypes include:

  - `MPI_CHAR`: Represents a character data type.
  - `MPI_INT`: Represents an integer data type.
  - `MPI_FLOAT`: Represents a floating-point data type.
  - `MPI_DOUBLE`: Represents a double-precision floating-point data type.
  - `MPI_LONG`: Represents a long integer data type.
  - `MPI_SHORT`: Represents a short integer data type.
  - `MPI_BYTE`: Represents a byte data type, which can be used for sending raw data.

### Message Matching

- The receive command and send command should match in terms of their src/dest send/receive tag, and communicator. If they don't match, the message will not be received and the program may hang or crash.

- A reciever can get a message without knowing:
  1. The amount of data in the message
  2. The sender of the message
  3. The tag of the message

- When the receiving order is hard to be maintained the MPI_recv can use a wildcard argument for:
  - source: `MPI_ANY_SOURCE`
  - tag: `MPI_ANY_TAG`

- the `MPI_Status`  is used as an arg for `MPI_Recv` to get that information about the message that was recieved.

- `MPI_Get_count` is used to get the number of elements received in a message.

```c
int MPI_Get_count(
    MPI_Status* status_p,
    MPI_Datatype type, 
    int* count_p
)
```

### Trapezoidal Rule in MPI

- Serial program:

```c
h = (b - a) / n;
approx = (f(a) + f(b)) / 2.0;
for (i = 1; i < n; i++) {
    x_i = a + i * h;
    approx += f(x_i);
}
approx *= h;
```

- We can parallelize this with MPI Psuedo code:

```c
get a, b, n;
h = (b - a) / n;
local n = n / num_procs;
local a = a + my_rank * n * h;
local b = a + n * h;
local_integral = Trap(local a, local b, local n, h);
if (my_rank != 0) {
    MPI_Send(&local_integral, 1, MPI_DOUBLE, 0, 0, MPI_COMM_WORLD);
} else {
    total_integral = local_integral;
    for (i = 1; i < num_procs; i++) {
        MPI_Recv(&local_integral, 1, MPI_DOUBLE, i, 0, MPI_COMM_WORLD, &status);
        total_integral += local_integral;
    }
    if (my_rank == 0) {
        printf("Total Integral: %f\n", total_integral);
    }
}
```

- We need to send a message to process 0 because we don't have the global variabes. So if we are not rank 0 we need to sent amessage to rank 0 with our local_integral. 

- Rank 0 will recieve the local_integral from all the other processes and add them together to get the total_integral.

- This means that rank 0 will have the result, so its the only one who can print it

- Full implementation:

```c

int main(void){
    int my_rank, comm_sz, n = 1024, local_n;
    double a = 0.0, b = 3.0, h, local_a, local_b;
    double local_int, total_int;
    int source;

    MPI_Init(NULL, NULL);
    MPI_Comm_rank(MPI_COMM_WORLD, &my_rank);
    MPI_Comm_size(MPI_COMM_WORLD, &comm_sz);

    h = (b - a) / n;
    local_n = n / comm_sz;
    local_a = a + my_rank * local_n * h;
    local_b = local_a + local_n * h;
    local_int = Trap(local_a, local_b, local_n, h);

    if (my_rank != 0) {
        MPI_Send(&local_int, 1, MPI_DOUBLE, 0, 0, MPI_COMM_WORLD);
        // We are sending our local_integral, 1 element, of type MPI_DOUBLE, to process 0 with tag 0 and using the MPI_COMM_WORLD communicator
    } else {
        total_int = local_int;
        for (source = 1; source < comm_sz; source++) {
            MPI_Recv(&local_int, 1, MPI_DOUBLE, source, 0, MPI_COMM_WORLD, MPI_STATUS_IGNORE);
            // We are recieving a message from the process source, we are recieving local_integral, 1 element, of type MPI_DOUBLE, with tag 0 and using the MPI_COMM_WORLD communicator. We are ignoring the status of the message.
            total_int += local_int;
            // We then take the local int and add it to total int (ONLY THREAD 0 CAN DO THIS)
        }
        // Only Rank 0
        if (my_rank == 0) {
            printf("With n = %d trapezoids, our estimate of the integral from %f to %f is %f\n", n, a, b, total_int);
        }
    }
    MPI_Finalize(); // We need to finalize the MPI environment when we are done using it.
    return 0;
}
```

### Inputs and Outputs of MPI Programs

- Most MPI implementations only allow process 0 in `MPI_COMM_WORLD` access to stdin.

- Process 0 must read the data (scanf) and send to the other processes.

- So, the assumption is only process 0 has access to stdin

```c
if(my_rank == 0){
    printf("Enter a, b, and n\n");
    scanf("%lf %lf %d", &a, &b, &n);
    for (dest = 1; dest < comm_sz; dest++){
        MPI_Send(&a, 1, MPI_DOUBLE, dest, 0, MPI_COMM_WORLD);
        MPI_Send(&b, 1, MPI_DOUBLE, dest, 0, MPI_COMM_WORLD);
        MPI_Send(&n, 1, MPI_INT, dest, 0, MPI_COMM_WORLD);
    }

}else{
    MPI_Recv(&a, 1, MPI_DOUBLE, 0, 0, MPI_COMM_WORLD, MPI_STATUS_IGNORE);
    MPI_Recv(&b, 1, MPI_DOUBLE, 0, 0, MPI_COMM_WORLD, MPI_STATUS_IGNORE);
    MPI_Recv(&n, 1, MPI_INT, 0, 0, MPI_COMM_WORLD, MPI_STATUS_IGNORE);
}
```

- This is one of the problems of send and receive.
- In this code we have process 0 read the data and then sends it to all the other processes.
- There must be someway to do communication a bit faster when it involves a large number of processes.

- This type of communication is called point to point communication. LEts talk about collective communication, which is when we want to send a message to all processes at once. This is called a broadcast.

### Collective Communication

1. Phase 1
    - Each process sends its data to a designated parent process.
2. Phase 2
    - The parent process collects data from its children and sends it to its parent.

- `MPI_Reduce` is a collective communication function that performs a reduction operation (such as sum, max, min, etc.) on data from all processes and returns the result to a designated root process.

```c

MPI_Reduce(
    void* input_data,
    void* output_data,
    int count,
    MPI_Datatype datatype,
    MPI_Op op,
    int dest_process,
    MPI_Comm comm
)

MPI_Reduce(&local_int, &total_int, 1, MPI_DOUBLE, MPI_SUM, 0, MPI_COMM_WORLD);

```

- Operations:
  - MPI_MAX: Maximum value
  - MPI_MIN: Minimum value
  - MPI_SUM: Sum of values
  - MPI_PROD: Product of values

- The difference between this and send/recv is that this has all the processes call the same collective function.

- For example, a program that attempts to match a call to MPI_Reduce on one process with a call to MPI_Recv on another process is errorneous and will crash. All processes must call the same collective function, and they must all call it in the same order.

- Example of where to use MPI_Reduce:

```c
local_int = Trap(local_a, local_b, local_n, h);
MPI_Reduce(&local_int, &total_int, 1, MPI_DOUBLE, MPI_SUM, 0, MPI_COMM_WORLD);
if (my_rank == 0) {
    printf("With n = %d trapezoids, our estimate of the integral from %f to %f is %f\n", n, a, b, total_int);
}
```

- In this example we use MPI_Reduce to sum up the local integrals from all the processes and store the result in total_int on process 0. This is much more efficient than using send and receive to collect the local integrals from all the processes.

- This prevents us from having to send multiple messages to process 0.
