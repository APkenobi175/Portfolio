# State Hoisting
* State Hoisting is taking the state and moving it up the tree into the app component. 
* To do this take where the states are defined `export function newAccountForm(){const[name, setName] = useState("")}` and bring them to the app.jsx and put it into the function App()
* Also `import { useState } form "react";
* ErrorMessage.jsx
* We want this to take in some props, look at the email and password, and name and validate and make sure they are all good
* Take your state values and pass them in as props to the error message. if the messages are good then it will return null.
* Hoisted or elevated the state up the component tree so you can pass the state into a function as a prop.

```jsx
//ErrorMessage.jsx
export function ErrorMessage(props){


    if (!props.email){
        return <div> Email cannot be blank! </div>
    }
    if (!props.name){
        return <div> Name cannot be blank! </div>
    }
    if (!props.password){
        return <div> Password cannot be blank! </div>
    }
    return null; // you can return null but not nothing
}
```
