## Refresher from last week

* We talked about the state design pattern and how to use it to implement a game.
* Each state gets its own script
  * Wander state
  * Attack state
  * Chase state
* We will have a state machine that will manage the transitions between states.
* The `_Process` will just call the current state's `_Process` method.
  * `_currentState.Process(delta);`
* You don't typically reuse states between different state machines, but its useful for us here you can use states across different entities.

* Our states are not nodes, just classes that have methods for handling the states.

* Your state could be nodes, and you just add/remove them when you transition between states.

## Wander State Implementation

* Every so often we're gonna pick a new direction and walk that direction, and then change direction again after a certain amount of time.

```csharp

public partial class WanderState : IEnemyState
{
    private Enemy enemy;
    private Vector2 direction;
    
    public void Enter()
    {
        direction = new Vector2(GD.RandRange(-1, 1), GD.RandRange(-1, 1)).Normalized();
        Timer timer = new Timer(); // Create a new timer
        timer.Timeout += () => { // This code will run every time the timer times out
            direction = new Vector2(GD.RandRange(-1, 1), GD.RandRange(-1, 1)).Normalized();
            enemy.Velocity = direction * 100; // Set the velocity to move in the new direction
        };
        timer.Autostart = true; // Start the timer immediately
        timer.WaitTime = 1; // wait 1 second before changing direction
        enemy.AddChild(timer); // add the timer as a child to the enemy.
        // There is no loop here, the timer will handle the timing for us. That is what autostart does, it will start the timer immediately and then it will call the timeout function every time the timer times out.
    }
}

```

* We create a timer that will call the timeout function every time it finishes. It will not be a one shot, meaning it will keep going and resetting itself every time it finishes.
* We set the velocity of the enemy to be in the direction we want to move, and then we let the physics engine handle the movement for us. We don't need to manually update the position of the enemy, we just set the velocity and let the physics engine do its thing.
* Now we want to detect when a player enters the enemy's detection radius, and then we want to transition to the chase state.

```csharp
//CHASE STATE

public partial class ChaseState : IEnemyState
{
    private Enemy enemy;
    private Character character;

    public void Enter(){

    }

    public void Exit(){

    }

    public void _Process(double delta)
    {
        if(character.Position.DistanceTo(enemy.Position) > 200) // if the character is outside of the detection radius
        {
            enemy.ChangeState(new WanderState()); // change back to the wander state
        }else if (character.Position.DistanceTo(enemy.Position) < 50)
        {
            enemy.ChangeState(new AttackState());
        }
        else{
            Vector2 direction = (character.Position - enemy.Position).Normalized(); // get the direction to the character
            enemy.Velocity = direction * 100; // set the velocity to move towards the character
            enemy.MoveAndSlide(); // move the enemy
        }
    }
}

```

* In the chase state, we check if the character is still within the detection radius. If they are not, we transition back to the wander state. If they are, we calculate the direction to the character and set the velocity to move towards them. We then call MoveAndSlide to move the enemy.

    ```csharp
using Godot;
    ```

```csharp
public partical class FishSpawner : Node2D

    }
```

### Seek Steering Behavior

* Thrust
  * Acceleration in the direction of the movement.
* Breaking
  * Deceleration in the direction of the movement.
* Steering
  * Acceleration in a direction that is not the direction of movement. This is what allows us to turn and change direction.

* All of our forces will be the same.

* Steering behaviors will be implemented as components.(scenes)
* Add a script to it ez
* We do want a interface for this because they're all gonna have a method to generate the steering force.
* We want to constrain our steering force to a max force so weird shit don't happen.

```csharp
using Godot;
using System;
using System.Collections.Generic;
public interface ISteeringBehavior
{
    Vector2 GetSteeringForce(List<Fish> neighbors);
}
```

```csharp

using Godot;
using System;

public partial class SeekSteeringBehavior : Node2D, ISteeringBehavior
{
    [Export]
    public Vector2 targetPosition = new Vector2(500, 400);
    public Vector2 GetSteeringForce(List<Fish> neighbors)
    {
        Fish fish = GetParent<Fish>();
        Vector2 desiredVelocity = targetPosition - fish.Position;
        Vector2 steeringForce = desiredVelocity - fish.Velocity;
        return steeringForce;
    }
}

private Vector2 truncate(Vector2 vector, float maxLength)
{
    if (vector.Length() > maxLength)
    {
        vector = vector.Normalized() * maxLength;
    }
    return vector;
}
```
